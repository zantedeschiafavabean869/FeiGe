# FeiGe UI 设计决策

## 决策表

| 项目 | 决策 |
| --- | --- |
| Surface | Command-Led 专业视频拆片工作台；核心流程为查看视频 → 拆镜 → 识别 → 整理剧本 → 导出。 |
| P1 | 当前视频、时间码、拆镜状态与分镜工作区。 |
| P2 | 当前项目、镜头筛选、AI 识别进度与剧本编辑。 |
| P3 | 项目管理、模型设置和导出。 |
| Action | 依据阶段突出一个主动作：自动拆镜、识别全部、生成剧本或导出全部。 |
| Signature | “帧门 / 切点轨道”：活动媒体与品牌使用克制的琥珀角标、时间码和短切点线。 |
| System | 石墨黑平面工作台；琥珀为主要操作与切点色，蓝色为信息，绿/黄/红为状态；Segoe UI Variable / Microsoft YaHei UI。 |
| Cliche budget | 0；删除紫青渐变、光晕、玻璃、霓虹与大圆角 SaaS 卡片。 |
| Content truth | 只呈现真实项目、视频帧、镜头与运行状态；不创建虚假统计和装饰性图表。 |
| Functional contract | 不改拆镜、识别、剧本、导出、设置、项目数据和 IPC 行为；保留 renderer 依赖的全部 ID、data 属性与 dialog 语义。 |

```text
│ FeiGe + 新建项目            │                                                         │

│ 任务状态                     │                                  │ 镜头列表或剧本编辑器      │
│ 中文 | EN        设置        │                                  │                         │

```

1366×768：

```text
┌── 侧栏 216 ──┬──────────── 项目栏（紧凑） ────────────┐
```

## 状态与交互

- 所有按钮有默认、hover、focus-visible、disabled、loading 状态；不使用 hover 位移。
- 状态必须同时有颜色与文字；Toast、进度与任务状态使用 `aria-live`。
- 标签页同步 `aria-selected`，场景筛选同步 `aria-pressed`。
- 支持 `prefers-reduced-motion`；长任务仍可取消，但显示为底部任务面板以保留上下文。
- 中文 / English 即时切换并本地记忆；仅翻译 UI，绝不翻译项目名、视频名、AI 内容、提示词、剧本与导出正文。

## 响应断点与检测

| 断点 | 行为 |
| --- | --- |
| ≥ 1800px | 侧栏 248px；视频 / 编辑器约 42 / 58；限制内容最大宽度避免 2560px 留白或过度拉伸。 |
| 1280–1799px | 侧栏 216–224px；视频 / 编辑器约 40 / 60；低频导出进入菜单。 |
| 1060–1279px | 上下堆叠工作面板，侧栏约 204px；各面板内部滚动。 |
| 高度 < 780px | 缩小页头、面板间距和视频控制区，保持主动作与首个镜头可见。 |

必须截图并检查 `1366×768`、`1440×900`、`1920×1080`、`2560×1080`；中英文均检查无横向滚动、按钮不截断、底栏可达、对话框不溢出。

## 主要失败风险

1. 英文长标签导致工具栏横向滚动：通过操作分组、导出菜单和 1366 英文截图检测。
2. 2560px 视频与镜头卡被无意义放大：通过最大宽度、比例列和超宽截图检测。
3. 高度较小时底部语言/设置或主动作消失：通过 1366×768 与 1440×700 检测。
4. 语言切换丢失播放位置、当前标签、滚动或未保存文本：用切换前后状态比对验证。
5. 动态进度或错误仍混入中文：用模拟进度、错误映射和英文模式全流程检查。
